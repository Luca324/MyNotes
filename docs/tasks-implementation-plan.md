# План реализации функционала задач (Tasks)

## База данных

### Текущая структура таблицы `notes`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER PRIMARY KEY | Уникальный идентификатор |
| `topic_id` | INTEGER NOT NULL | ID темы, к которой привязана заметка |
| `title` | TEXT | Заголовок заметки (опционально) |
| `content` | TEXT | Содержимое заметки |
| `order_index` | REAL NOT NULL DEFAULT 0 | Порядок сортировки |
| `created_at` | INTEGER | Unix timestamp создания |

### Изменения в структуре БД

**Добавить поля в таблицу `notes`:**
- `is_task BOOLEAN DEFAULT 0` - флаг, является ли запись задачей
- `done BOOLEAN DEFAULT 0` - флаг выполнения задачи (актуально только для задач)

**SQL миграция:**
```sql
ALTER TABLE notes ADD COLUMN is_task BOOLEAN DEFAULT 0;
ALTER TABLE notes ADD COLUMN done BOOLEAN DEFAULT 0;
```

**Обновленная структура таблицы `notes`:**
```sql
CREATE TABLE IF NOT EXISTS notes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  topic_id INTEGER NOT NULL,
  title TEXT,
  content TEXT,
  order_index REAL NOT NULL DEFAULT 0,
  created_at INTEGER DEFAULT (strftime('%s', 'now')),
  is_task BOOLEAN DEFAULT 0,
  done BOOLEAN DEFAULT 0,
  FOREIGN KEY (topic_id) REFERENCES topics (id) ON DELETE CASCADE
);
```

## Детали реализации

### Виртуальная тема "Выполненные задачи"

**Концепция:**
- Не хранится в БД как отдельная тема
- Создается динамически в UI при наличии выполненных задач
- Визуально выглядит как сворачиваемая тема
- При разворачивании показывает список выполненных задач

**Реализация:**
```typescript
// В TopicContentComponent
const completedTasks = tasks.filter(task => task.done)
const hasCompletedTasks = completedTasks.length > 0

// Рендерим виртуальную тему
{hasCompletedTasks && (
  <CompletedTasksGroup 
    tasks={completedTasks}
    onToggleTask={toggleTaskDone}
  />
)}
```
