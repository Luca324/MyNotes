# Архитектура проекта MyNotes

## Общее описание

MyNotes - это мобильное приложение для заметок с рекурсивной иерархической структурой. Основная фича - возможность создавать темы (папки) и подтемы любой глубины, а также заметки внутри каждой темы.

## Технологический стек

- **Framework**: React Native с Expo
- **Навигация**: Expo Router (file-based routing)
- **База данных**: SQLite (expo-sqlite)
- **Язык**: TypeScript + JavaScript
- **Версия Expo**: ~53.0.23
- **Версия React**: 19.0.0
- **Версия React Native**: 0.79.5

## Основные концепции

### 1. Рекурсивная структура тем (Topics)

Темы могут иметь родительские темы (`parent_id`), что позволяет создавать иерархию любой глубины:
- Тема уровня 0 (корневая) - `parent_id = null`
- Подтема уровня 1 - `parent_id = <id корневой темы>`
- Подтема уровня 2 - `parent_id = <id подтемы уровня 1>`
- И так далее...

### 2. Заметки (Notes)

Заметки привязаны к конкретной теме через `topic_id`. Каждая заметка может иметь:
- `title` - заголовок (опционально, может быть `null`)
- `content` - содержимое (может быть `null`)
- `order_index` - порядок сортировки
- `created_at` - дата создания (Unix timestamp)

**Особенности работы с заметками**:
- Автоматическое создание при первом вводе текста в редакторе
- Автоматическое обновление при каждом изменении
- Автоматическое удаление пустых заметок, если пользователь передумал создавать

### 3. Вкладки (Tabs)

Вкладки - это закрепленные темы для быстрого доступа. Пользователь может закрепить любую тему как вкладку.

**Специальная вкладка "All"**:
- Вкладка с `id: 0` всегда присутствует в списке
- Показывает все корневые темы (без родителя)
- Нельзя удалить или открепить
- Используется как точка входа в приложение

**Функциональность**:
- Переключение между вкладками через свайпы (влево/вправо)
- Закрепление/открепление тем через меню настроек

## Паттерны и подходы

- **File-based routing**: Используется Expo Router для навигации
- **Context API**: AppContext для глобального состояния (текущая тема и вкладки)
- **Custom Hooks**: useNotes, useModal для переиспользования логики
- **Service Layer**: databaseService.ts для абстракции работы с БД
- **TypeScript**: Типизация основных сущностей (Topic, Note, Tab)
- **Автоматическое обновление**: Компоненты автоматически обновляют данные при разворачивании/монтировании
- **Оптимизация**: Использование `useMemo` для вычисляемых значений, `useRef` для отслеживания состояния
- **Жесты**: Поддержка свайпов для переключения между вкладками
- **Обработка клавиатуры**: Автоматический padding при открытии клавиатуры

## Цветовые схемы

Проект использует функции из `colorSchemes.js`:
- `getDepthColor(depth)` - возвращает цвет фона в зависимости от глубины вложенности
- `getTextColor(depth)` - возвращает цвет текста в зависимости от глубины вложенности

Это позволяет визуально различать уровни вложенности тем.

## Новые функции

### Свайпы для переключения вкладок
- Свайп влево - переход к следующей вкладке
- Свайп вправо - переход к предыдущей вкладке
- Циклическое переключение (после последней - первая, после первой - последняя)

### Автоматическое управление заметками
- Создание заметки при первом вводе текста
- Автоматическое обновление при каждом изменении
- Удаление пустых заметок, если пользователь передумал

### Улучшенная работа с вкладками
- Функция открепления вкладок (`removeTab`)
- Специальная вкладка "All" для просмотра всех корневых тем
- Автоматическое обновление списка вкладок при изменениях
